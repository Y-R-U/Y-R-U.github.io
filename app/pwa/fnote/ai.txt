Fast Notes PWA - AI Reference File
====================================

## File Structure
app/pwa/fnote/
  index.html          - Single HTML file with all markup + inline CSS (dark/light themes via CSS vars)
  manifest.json       - PWA manifest with inline SVG icon
  sw.js               - Service worker (fnote-v2), cache-first; skips /api/ routes
  ai.txt              - This file
  functions/
    api/
      notes.js        - Cloudflare Pages Function: GET/POST /api/notes?user=<username>
                        Reads/writes from KV binding FNOTE_DATA under key "fnote:<username>"
                        Keeps KV credentials server-side — frontend never sees them
  js/
    app.js            - Bootstrap / entry point (checks for username; shows login or app)
    auth.js           - Username helpers: get/set/clear/sanitize/validate (localStorage key: fnote_username)
    store.js          - localStorage CRUD layer + cloud sync (initSync, syncFromCloud, _pushToCloud)
    router.js         - Hash-based routing
    home.js           - Home screen: card grid, breadcrumbs, new-note flow
    editor.js         - Note editor: contentEditable, toolbar, auto-save
    settings.js       - Settings popup (theme toggle, Account/switch-user section, export/import)
    modal.js          - Custom modal/popup system (replaces alert/prompt/confirm)
    theme.js          - Dark/light theme switching
    utils.js          - Shared utilities

## localStorage Schema
- `fnote_username`: string — the chosen username (set at login, cleared on switch)
- `fnote_items`:    JSON array of items:
    { id, type:"note"|"folder", title, content (HTML string),
      parentId (null=root), createdAt, updatedAt }
- `fnote_settings`: { theme: "dark"|"light" }

## Cloud Storage (Cloudflare KV)
- KV namespace: FNOTE_DATA  (bound to Cloudflare Pages project as env var FNOTE_DATA)
- Keys: "fnote:<username>"  →  JSON string:  { "items": [...] }
- API endpoint exposed by Pages Function:
    GET  /api/notes?user=<username>  →  { items: [...] }
    POST /api/notes?user=<username>  body: { items: [...] }  →  { ok: true }
- Username validation: /^[a-z0-9_-]{2,30}$/

## Sync Strategy
1. On login / app start: syncFromCloud() fetches remote items and merges with local.
   Merge rule: for each item id, keep the version with the higher updatedAt timestamp.
2. On every local write (save / remove / importAll): _pushToCloud() fire-and-forgets the full items array.
3. Offline: writes succeed locally; next load re-syncs and pushes accumulated changes.

## Routing (hash-based)
- #/              → Home (root items)
- #/folder/{id}   → Home (folder contents)
- #/note/{id}     → Note editor

## Views
- login   - shown first-visit; username input panel; sets fnote_username then calls initApp()
- home    - card grid (includes sync-dot status indicator top-left)
- editor  - note editor

## Functions by File

### js/auth.js
- getUsername()           - returns stored username or null
- setUsername(name)       - saves to localStorage
- clearUsername()         - removes from localStorage (triggers reload → login screen)
- sanitizeUsername(raw)   - strips invalid chars, lowercases, max 30
- isValidUsername(name)   - returns bool; /^[a-z0-9_-]{2,30}$/

### js/store.js  (new exports)
- initSync(username)      - call once after login; tells store which user to push to
- syncFromCloud()         - async; fetches, merges, pushes back if needed; returns merged array
  (all original exports unchanged: getAll, getById, getChildren, getBreadcrumbs, save, remove,
   create, getSettings, saveSettings, exportAll, importAll)

### js/settings.js  (signature change)
- openSettings(username, onRefresh)  - username shown as @user with Switch User button

### js/app.js
- Entry point. Checks getUsername():
    - No username → initLogin() → shows login view, wires input + button
    - Has username → initApp(username) → initSync, initTheme, router, syncFromCloud

### functions/api/notes.js  (Cloudflare Pages Function)
- onRequestGet    - GET  /api/notes?user=<u>  returns KV data or {"items":[]}
- onRequestPost   - POST /api/notes?user=<u>  writes body to KV
- onRequestOptions - CORS preflight

## Key Design Notes
- contentEditable + document.execCommand for rich text (paste strips formatting)
- Checkbox/radio use custom spans with data-checked, contenteditable="false" on toggles
- Auto-save debounced at 800ms with "Saved" flash indicator
- Editor toolbar has two copy buttons (right-aligned via .tool-copy-group / margin-left:auto):
    #tool-copy-text  — copies note content as plain text; .fn-cb → [ ]/[x], .fn-rd → ( )/(*)
    #tool-copy-html  — copies contentEl.innerHTML verbatim
    Both flash their border accent colour for 800ms as visual confirmation
- Cards grid: 3 columns on mobile, auto-fill on wider screens
- Folders can nest inside folders (parentId chain)
- Delete button acts as cancel for new empty notes
- sync-dot in home header top-left: grey=idle, blue-pulse=syncing, green=synced, red=error

## User Header Chip & Recent Usernames
- #btn-user pill button in home header (top-right, left of ⚙) shows @currentUsername
- Clicking it opens #user-popup floating card with:
    1. @currentUsername header
    2. "Switch to" recent-users list (other usernames from history):
       - Click a name → setUsername(name) + reload (no confirm; deliberate click)
       - Click × → removeRecentUsername(name) + re-render the list
    3. "Switch User" button → modalConfirm then clearUsername() + reload (enter any username)
- Recent usernames stored in localStorage key `fnote_recent_users` (JSON array, max 10)
  - Managed in auth.js: getRecentUsernames(), removeRecentUsername(name)
  - setUsername() auto-adds to recents (deduped, most-recent first)
- Popup closes on any outside click

## Drag & Drop (home.js)
- Pointer Events API; different initiation strategy per pointer type:
  - Mouse: drag activates once pointer moves > DRAG_THRESHOLD (10px) after pointerdown
  - Touch: long-press (500ms, LONG_PRESS_MS) required before drag activates;
    if finger moves > DRAG_THRESHOLD before timer fires, timer is cancelled and
    the gesture is treated as a scroll (no drag initiated)
- `touch-action: pan-y` on .card normally; `touch-action: none` via body.drag-active once drag is live
- .card.long-press-pending class added on touch pointerdown → pulsing accent border gives
  visual feedback that the hold is being recognised; removed when timer fires or is cancelled
- Ghost label (position: fixed, pointer-events: none) follows pointer once drag is active
- #drag-cancel button appears (top-left) during drag; breadcrumb links become drop zones
